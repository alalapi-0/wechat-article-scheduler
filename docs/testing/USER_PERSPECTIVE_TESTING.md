# 用户视角测试

Codex 可在明确指定的“用户视角审计”角色中模拟不懂编程的公众号作者。该角色只复现流程、记录证据、输出问题和改进任务，不在同一次审计中修改代码；Cursor/推进 Agent 再按任务落地修复，避免测试者同时改写被测对象。

## 审计问题

1. 我能否立刻知道现在会不会真实发布？
2. 下一步操作是否明显？
3. 操作后是否知道成功了什么？
4. 失败时是否知道原因、影响和恢复方式？
5. 普通视图是否泄露内部字段或技术噪声？

## 方法

- 使用真实浏览器走完整用户路径。
- 默认 mock；若验证 real_api，必须按 `REAL_API_TESTING.md` 获得明确授权。
- 检查页面、console、network、响应式和数据持久化。
- 每个问题记录复现步骤、期望、实际、严重度、截图/日志位置和建议验收条件。
- 不以单张截图或静态 HTML 作为成功证据。

输出任务交给实现 Agent 后，修复仍需重新运行浏览器测试。
