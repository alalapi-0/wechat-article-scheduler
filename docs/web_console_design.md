# Web 控制台设计（轻量）

## 目标

在不引入 React/Vue 的前提下，提供本地可用、可审计的操作面板。

## 当前现状

- 已实现：`FastAPI + 内嵌 HTML`，支持状态查看与 scan/plan/run-once 触发
- 部分实现：文章、任务、事件基础查询
- 未实现：鉴权、多用户、复杂筛选、审批工作流

## 设计原则

- 继续使用服务端模板或极简原生 JS
- Web 仅作为 CLI 的可视化壳，业务逻辑仍在 Python 服务层
- 默认只允许本机访问（`127.0.0.1`）

## 计划接口

- 只读：状态、文章、任务、事件
- 受控写：scan、plan、run-once（后续增加审批闸门）
- 明确禁止：直接触发无审计的真实发布批处理
