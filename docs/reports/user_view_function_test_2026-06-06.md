# 用户视角功能测试报告

测试时间：2026-06-06 13:24 CST

## 1. 项目理解

- 项目名称：wechat-article-scheduler
- 项目目标：个人本地微信公众号草稿、排期、发布与外部 Browser Agent 任务包工作台。
- 目标用户：需要批量管理微信公众号文章的个人运营者。
- 核心流程：本地文章导入 -> 排期 -> 到点创建/更新草稿 -> 可选发布 -> 事件/proof。
- 当前测试入口：CLI、FastAPI Web 工作台、微信公众号官方 API。
- 是否需要真实 API：真实草稿、远端同步和真实发布需要；普通开发默认 mock。
- 本次是否使用真实 API：是。只创建并删除 1 篇测试草稿，没有正式发布。

当前项目只把“本项目自己创建的草稿记录”写进本地数据库，没有把公众号已有草稿和已发布文章同步回来。当前“定时发布”是本地 scheduler 到点调用 API，不是把定时时间写进微信公众号后台。

## 5. 问题清单

### P1-001：没有公众号远端内容同步层

- 模块：Wechat API / 数据模型 / Web
- 问题：只能显示本项目写入的 `wechat_drafts`，看不到公众号已有草稿和已发布内容。
- 影响：无法把现有 100 篇草稿纳入排期、编辑、删除和状态追踪。
- 建议：实现 `draft/batchget`、分页同步、远端稳定 ID、本地镜像表和只读同步页面。

### P1-002：已发布列表接口无权限且项目没有能力探测

- 复现：调用 `freepublish/batchget`。
- 实际：当前账号返回“当前 API 未授权”。
- 影响：不能可靠展示或删除已发布文章。
- 建议：新增 capability probe；未授权时显示“账号接口权限不足”，绝不能显示为空列表。

### P1-003：真实 draft-only 每周续排会重复同一批文章

- 复现：100 篇 imported，第一周生成 35 个任务并将任务标记 done，但文章保持 imported；一周后再次 plan。
- 实际：第二周 35 篇与第一周重合 35 篇。
- 预期：顺延到第 36-70 篇。
- 原因：计划器只排除 pending，draft-only 完成没有独立终态/游标。
- 建议：引入 `remote_draft_ready`、`scheduled_local`、`submitted` 等状态和唯一约束。

### P1-004：mock 仅草稿被错误标记为已发布

- 复现：mock + `publish_action=draft` 到点执行。
- 实际：文章详情显示“已发布”，`stats.drafted=0`。
- 原因：`MockWechatAdapter.submit_publish()` 忽略 `force=False`，没有返回 `skipped`。
- 影响：测试环境掩盖真实 draft-only 状态错误。

### P1-005：空 Markdown 被正常入库

- 实际：标题取文件名 `empty`，正文为空，但 preflight 仍 `ready=true`，可继续安排时间。
- 影响：批量周计划会混入空稿。
- 建议：上传/scan 阶段阻止空正文；preflight 将空正文设为 blocking。

### P1-006：没有固定合集和后台设置的批量模型

- 实际：批量发布设置只有时间、发布方式、留言、作者、原文链接。
- 缺失：固定合集、推荐/通知类设置、后台定时设置、设置继承策略。
- 建议：先把字段加入任务目标模型，并逐项标记 `api_supported/browser_required/unavailable`。

### P1-007：远端批量删除尚无安全实现

- 草稿：官方单篇删除已验证，可对稳定 `media_id` 做受控批处理。
- 已发布：列表权限当前未授权，删除能力必须先探测。
- 风险：按标题删除会误删。本次后台已有两篇同标题测试稿，标题匹配保护成功阻止了删除。
- 建议：两阶段删除、稳定 ID、影响预览、确认清单、逐条审计、限速、失败可重试。

### P1-008：外部 Agent 任务包会为 mock 草稿生成真实后台操作指令

- 实际：任务包包含 `mock_media_*`，却要求打开公众号后台定位草稿。
- 建议：mock 任务包明确标记 simulation，或禁止导出为真实后台任务。

### P2-001：普通视图暴露本地 inbox 绝对路径

- 实际：首页普通视图显示 `/tmp/.../inbox`。
- 建议：路径移入高级信息。

### P2-002：`python`/`python3` 文档不一致

- 实际：AGENTS 命令使用 `python`，当前环境报 command not found；README 使用 `python3`。
- 建议：统一使用 `.venv/bin/python` 或 `python3`。

### P2-003：真实 API dry-run 阻塞时仍输出清理真实草稿提示

- 实际：缺少凭证、没有创建草稿，仍提示后台会留下草稿。
- 建议：仅在 `success_count > 0` 时显示清理提示。

### P2-004：日期控件依赖外部 CDN

- 实际：页面从 jsDelivr 加载 flatpickr。
- 影响：本地离线工作台可能失去日期选择体验；浏览器 Agent 对 readonly 输入也更难操作。
- 建议：本地打包静态资源或提供原生 datetime-local 回退。

## 8. 体验不舒服但不一定是 bug 的地方

- “仅草稿”完成后出现“已发布”，用户无法判断真实状态。
- 批量设置没有合集，无法完成固定栏目工作流。
- 页面能设置“每篇间隔”，但不能表达“每天 5 篇、只排未来 7 天、下周自动续排”。
- 远端草稿与本地草稿没有明确分区。
- 已发布接口无权限时缺少产品化解释与替代路径。
- 删除应先显示将删除的稳定 ID、标题、更新时间和数量。
- 外部 Agent 任务包不能把 mock 草稿伪装成真实后台任务。

## 9. 给推进轮 Agent 的修改任务

### 任务 1：实现公众号远端只读同步

- 目标：读取全部远端草稿，按分页同步到本地。
- 涉及：real adapter、数据库 migration、CLI、Web。
- 数据字段：`remote_type`、`media_id/article_id`、`title`、`update_time`、`last_seen_at`、`content_hash`、`sync_status`、`raw_summary_redacted`。
- 验收：当前账号同步 5 篇草稿；不读取/打印 token；重复同步不重复入库。
- 优先级：P1。

### 任务 2：实现已发布内容 capability probe

- 目标：探测 `freepublish/batchget` 权限并区分“无权限/空列表/请求失败”。
- 验收：当前账号显示明确“未授权”，不显示“0 篇”。
- 后续：仅在列表权限通过后评估 `freepublish/delete`。
- 优先级：P1。

### 任务 3：让现有远端草稿进入本地排期

- 目标：选择远端 `media_id`，不重复创建草稿，到点直接提交或生成后台辅助任务。
- 修改：publish job 增加 `remote_media_id`、`source_kind=remote_draft`。
- 验收：已有远端草稿排期时不调用 `draft/add`。
- 优先级：P1。

### 任务 4：实现每周续排策略

- 配置示例：`days=7`、`max_per_day=5`、`preferred_hours=[8,11,14,17,20]`、固定合集、每周运行时间。
- 必须有持久游标和唯一约束，不能按 imported 状态猜进度。
- 验收：100 篇连续执行三周，周计划分别选 1-35、36-70、71-100，零重复。
- 优先级：P1。

### 任务 5：修正 draft-only 状态机和 mock 语义

- 状态建议：`remote_draft`、`scheduled_local`、`submit_pending`、`publishing`、`published`、`failed`。
- 验收：仅创建草稿时不显示“已发布”；mock 与 real 行为一致。
- 优先级：P1。

### 任务 6：实现远端草稿批量删除

- 默认 read-only；删除功能单独开关。
- 流程：同步快照 -> 勾选稳定 media_id -> 影响预览 -> 生成 delete manifest -> 二次确认 -> 逐条 `draft/delete` -> 重新同步。
- 限制：禁止按标题直接删除；默认并发 1；记录每项结果；失败可重试但不重删成功项。
- 验收：对 3 篇专用测试草稿删除 2 篇，未选稿保持不变。
- 优先级：P1。

### 任务 7：实现已发布文章删除的权限门控

- 先验证列表权限和删除接口权限；当前账号默认禁用。
- UI 必须说明删除不一定等于撤回已送达消息，语义以官方接口实际返回为准。
- 验收：无权限账号按钮禁用并说明原因；有权限测试账号只删除专用测试文章。
- 优先级：P1。

### 任务 8：建立固定合集与后台字段能力模型

- 字段至少包括：合集、留言、仅粉丝留言、推荐/通知、封面显示、后台定时。
- 每个字段声明：API 支持、浏览器辅助、不可用、需要人工确认。
- 验收：批量设置可选择固定合集；任务包完整携带目标值。
- 优先级：P1。

### 任务 9：修复输入与任务包问题

- 阻止空正文入库/排期。
- mock 草稿任务包不得要求在真实后台定位。
- API dry-run 清理提示按实际成功数显示。
- 优先级：P1/P2。

### 任务 10：增加远端自动化审计与恢复

- 每次周计划、同步、更新、删除生成不可变运行记录。
- 保存 selected IDs、请求结果、失败原因和重试状态，不保存 token/cookie。
- 提供 `--dry-run`、`--resume-run-id`、`--max-items` 和 emergency stop。
- 优先级：P1。

## 新 Agent 实施顺序

建议拆成四轮，不要一次完成：

1. Round A：远端只读同步 + capability probe。
2. Round B：远端草稿本地排期 + 每周续排状态机。
3. Round C：固定字段模型 + Web 批量工作台。
4. Round D：远端批量删除 + 审计/恢复；已发布删除仅在权限通过后启用。

每轮都先更新 `docs/rounds.md`，再同步 `scripts/agent_gate.py` 与 `tests/test_agent_gate.py`。真实 API 测试只使用专用 `[API-TEST]` 内容，默认 `WECHAT_ENABLE_PUBLISH=false`，禁止删除非测试远端内容。
