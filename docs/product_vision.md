# 产品愿景

Status: Current P0 product direction

## 主定位

本项目当前定位为：**个人本地微信公众号发布工作台**。

它是给我自己使用的本地工具，不是商业 SaaS，不是多平台发布器，也不是团队协作后台。当前最重要的目标是把微信公众号文章发布闭环做小、做稳、做可执行。

## 阶段一：个人本地微信公众号发布工作台

阶段一是 P0。阶段一完成前，不开发其他平台。

阶段一要打通微信公众号文章发布全流程：

- 文章导入：支持本地 Markdown、TXT、HTML、小说章节、长文文章。
- 文章查重：按内容 hash、标题规范化等方式避免重复导入。
- 文章预览：在 Web 控制台查看原文、发布正文、摘要和预检结果。
- Markdown 到微信公众号 HTML：生成适合 `draft/add` 的保守内联样式 HTML。
- 段落样式：处理标题、段落、空行、列表、图片和基础强调样式。
- 摘要 120 字限制：scan、预览和发布前兜底都必须遵守。
- 封面管理：支持默认封面、单篇封面、封面素材上传和封面路径记录。
- 方形/横向封面预览：在发布前看到公众号常见比例效果。
- 微信草稿创建：通过官方 API 创建草稿，mock 模式默认不联网。
- 微信草稿更新：在草稿已存在时支持更新或至少提供明确替代流程。
- 本地计划发布时间：为文章生成 `scheduled_at`，支持手动调整。
- 本地 scheduler：按本地时间处理到期任务，支持失败重试。
- Web 控制台：提供 Dashboard、文章列表、文章详情、预览、草稿、队列、设置和事件日志。
- 失败重试：记录错误原因、重试次数、下次重试时间和人工处理建议。
- 事件日志：记录扫描、导入、排期、草稿、发布、失败和人工确认。
- 人工确认：真实发布和 browser_assist 流程必须有人在环。
- browser_assist：仅作为微信公众号 API 无法覆盖字段时的本地自用补充路线。

阶段一的安全边界：

- 默认 `WECHAT_MODE=mock`。
- `WECHAT_MODE=real` 是显式真实 API 测试模式，允许验证真实草稿创建和正式发布接口。
- 需要草稿-only 时，显式设置 `WECHAT_ENABLE_PUBLISH=false`。
- 真实正式发布仍必须由任务级选择“正式发布”触发；任务级“仅草稿”不会调用发布接口。
- 不读取或打印 `.env` 密钥。
- 不保存公众号后台密码。
- 不保存公众号后台 cookie。
- 不绕过验证码、扫码登录或平台风控。
- browser_assist 不默认点击最终发布。

阶段一的定时发布分两类：

- 当前实现：本地 scheduler 根据 `publish_jobs.scheduled_at` 到点调用微信 API。
- 待核验/后备：如果要把发布时间直接设进微信后台草稿箱，需先确认官方 API 是否支持；不支持时走 browser_assist + 人工确认。

## 阶段二：文本平台扩展

阶段二在微信公众号闭环稳定后再考虑，当前不开发。

候选平台：

- 知乎
- 豆瓣
- 个人博客 / 静态站
- WordPress

推荐顺序是先做 manual_export，再评估 browser_assist 或 official API。阶段二启动前必须先做新的治理轮，确认微信公众号主线不受影响。

## 阶段三：图文/视频平台扩展

阶段三当前不开发。

候选平台：

- 小红书
- 微信视频号
- Bilibili
- 抖音
- 快手

这些平台通常涉及更强风控、视频资产、封面、标签、审核和登录态问题。阶段一不得为这些平台提前实现 adapter 或数据库模型。

## 阶段四：音频/音乐平台扩展

阶段四当前不开发。

候选平台：

- 网易云音乐
- 播客平台
- 喜马拉雅或其他音频平台

这类平台涉及音频文件、版权、歌词、节目元数据和审核流程，只保留为长期 backlog。

## 非目标

- 阶段一完成前，不开发其他平台。
- 不把项目做成商业 SaaS。
- 不引入多租户、团队权限、计费系统。
- 不引入大型前端框架作为当前阶段依赖。
- 不为了未来多平台抽象破坏当前微信公众号 CLI 链路。
- 不默认开启真实发布。
