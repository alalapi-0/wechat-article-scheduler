# 产品愿景：本地微信公众号文章发布工作台

## 定位

本项目不是“全自动发布机器人”，而是**本地优先、可审计、可人工把关**的微信公众号发布工作台。

## 目标用户

- 独立内容创作者
- 小团队新媒体运营
- 需要本地可控流程、暂不依赖复杂 SaaS 后台的组织

## 核心价值

- 以本地文件为源，降低内容资产漂移风险
- 在 mock/real 两套模式下复用同一套 CLI 工作流
- 通过事件审计追踪每次扫描、排期与发布动作
- 先保障“可控与安全”，再逐步增强“效率与自动化”

## 非目标（当前阶段）

- 不做重前端单页应用（React/Vue）
- 不做无人值守的完全自动发布
- 不做网页登录公众号后台模拟
- 不做复杂封面智能裁剪流水线

## 分阶段演进（摘要）

- Round 1（当前）：治理基线、文档重构、摘要与渲染最小修复
- Round 2~4：内容库、调度域、数据库迁移与 Web 控制台设计落地
- Round 5~7：安全闸门、审核流、可观测性与运维加固
- Round 8~9：适度产品化与可用性完善
