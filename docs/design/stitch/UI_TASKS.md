# Stitch UI 设计任务模板

以下模板已按微信公众号本地工作台调整。每次使用时复制到 `prompts/` 或任务说明中，并删除不适用项。

## 1. 主控制台 Dashboard

**目标**：让个人用户快速知道当前是否安全、下一步操作、最近是否成功。

**必须包含**：

- mock / dry-run / real API / 草稿-only 顶部状态；
- “找文章 -> 安排时间 -> 执行到点文章”主流程；
- 文章、任务队列、草稿和最近事件摘要；
- 最近成功、失败任务与恢复建议；
- 进入文章列表、发布队列、草稿页和高级信息的入口。

**边界**：桌面信息密度优先；数据库路径、原始 JSON 和内部统计默认隐藏。

## 2. 真实 API 草稿任务页

**目标**：配置并观察一次真实微信公众号草稿创建或更新，不把 mock 冒充 real。

**必须包含**：

- 文章、封面、摘要、合集和发布动作摘要；
- provider 固定为微信公众号，展示当前账号能力结论；
- 参数与预检结果；
- pending / running / success / partial / failed 状态；
- 错误原因、影响、重试和返回安全模式；
- 草稿结果、远端 `media_id` 只放详情/高级信息；
- 保存、重试和人工确认，禁止默认触发最终发布。

## 3. 审核台 Review Workbench

**目标**：核对文章正文、封面、摘要、草稿字段和 proof，再决定通过、退回或移出流程。

**必须包含**：

- 正文与公众号近似预览；
- 封面横向/方形预览；
- mock / dry-run / real_api / draft-only 标记；
- 通过、退回、移出流程；删除必须二次确认；
- 普通视图的人话错误和高级区的路径、日志、元数据；
- 外部 Agent 任务包和 proof 状态。

## 4. Debug / Observability 页面

**目标**：帮助开发者与 Agent 排查，不污染普通用户首屏。

**必须包含**：

- API health、scheduler health 和数据库状态；
- console/network 错误摘要；
- 任务队列、claim、重试和卡住任务；
- 文件扫描、草稿同步和最近操作日志；
- 过滤、复制脱敏摘要、跳转到关联文章；
- 所有 token、secret、cookie 永不展示。

## 非当前主线模板

视觉结果审核台、游戏资产 Gallery、视频生成 Workbench 不属于当前微信公众号 P0，不应成为默认导航或当前路线任务。若未来 backlog 正式启用图片/视频/游戏资产项目，再使用 `PROMPT_TEMPLATES.md` 中对应模板，并先更新权威路线图。
