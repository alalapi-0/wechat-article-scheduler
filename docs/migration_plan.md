# 迁移计划（保守演进）

## 目标

在不破坏现有 CLI 闭环的前提下，将项目逐步演进为“本地微信公众号文章发布工作台”。

## 迁移原则

- 小步、可回滚、可测试
- 先文档和治理，再代码拆分
- 保持 `scan/plan/run-once` 命令稳定

## 阶段计划

### 阶段 A（已完成）

- 建立治理文档基线与能力审计
- 目录骨架到位（content/scheduler/renderers/web/cover_assets）
- digest 120 统一与上传前兜底

### 阶段 B（下一步）

- 抽离调度域逻辑，补充领域测试
- 增强事件结构化字段
- 增加人工审核闸门（默认不自动发布）

### 阶段 C（后续）

- 内容库与封面资产的元数据模型
- Web 控制台只读仪表盘与受控写操作
- 迁移脚本体系（版本表、前向迁移、回滚策略）

## 风险与缓解

- 风险：目录骨架与旧模块同名引发导入冲突  
  缓解：保留兼容层并加入回归测试
- 风险：真实发布误触发  
  缓解：默认 mock，建议 real 模式下先关闭 `WECHAT_ENABLE_PUBLISH`
