# 未来 40 轮开发路线图

Status: Backlog / Not current development target

本路线图服务参考架构吸收后的长期推进。它不是当前已实现能力声明。每轮都必须保护现有微信公众号草稿链路，真实发布默认关闭。
## Round 1：参考架构吸收治理轮
- 目标：吸收 TryPost / Mixpost / BrightBean / browser_mcp 参考模式，更新文档、协议、路线图、数据模型，不做大规模功能开发。
- 本轮要做的具体任务：
  - 创建 reference_absorption 文档组。
  - 新增 absorbed_methodology。
  - 新增长期数据模型、适配器、调度、review/proof、manifest 和平台矩阵文档。
  - 同步 repo_protocol、rounds、agent_gate 和测试。
  - 创建轻量目录/接口骨架。
  - 阅读与本轮相关的现有代码和文档，确认 参考架构吸收治理轮 的真实边界。
- 交付物：
  - 与“参考架构吸收治理轮”对应的代码、文档或配置更新。
  - 聚焦测试或 smoke 记录。
  - 必要的用户/Agent 交接说明。
- 验收标准：
  - `scan/plan/run-once` 不回归。
  - 默认 mock 或 draft-only 安全边界保持。
  - 本轮文档与实现一致，不声称未验证平台已支持。
  - 相关测试或 smoke 通过；失败时有明确原因和下一步。
- 不做什么：
  - 不做完整多平台发布、不 fork 参考仓、不引入重依赖、不重构微信链路。
- 风险边界：
  - 任何涉及真实平台的能力必须有显式开关、dry-run、审计和回滚说明。
  - browser_assist 不得绕过验证码、扫码、风控或自动点击最终发布。
- 是否允许改数据库：否。
- 是否允许真实发布：否。
- 依赖前置轮次：无。
